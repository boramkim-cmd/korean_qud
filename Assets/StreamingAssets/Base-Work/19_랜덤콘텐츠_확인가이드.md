# 랜덤 콘텐츠(절차적 생성) 확인 시나리오

**작성일시:** 2026-01-13 11:45
**목적:** 작업한 Naming.xml과 HistorySpice.json의 실제 작동 확인

## 1. 랜덤 NPC 이름 (Naming.xml)

게임은 `Prefix`(접두사) + `Infix`(중접사) + `Postfix`(접미사)를 조합해 이름을 만듭니다.

### 테스트 방법
1. 콘솔(`) 열기
2. `wish:spawn WatervineFarmer` 입력 (5~6번 반복하여 여러 명 소환)
3. `L` 키를 눌러 이름 확인

### 정상 작동 예시
- **파유멧** (Fa-yu-met)
- **쿠라슘** (Ku-ra-shum)
- **아기르** (A-gi-r)
- **메멧** (Me-met)

> **✨ 체크포인트:** 영어 이름이 아니라 **한글 이름**으로 생성되어야 합니다.

---

## 2. 랜덤 지역 이름 (Naming.xml - Site)

"바나나 숲", "폐허" 같은 템플릿 뒤에 랜덤한 쿼드식 이름이 붙습니다.

### 테스트 방법
**방법 A: 텔레포트**
1. 콘솔(`) 열기
2. `wish:teleport BananaGrove` 입력
3. 도착 후 현재 위치 이름 확인

**방법 B: 월드맵 탐험**
1. 조파 마을에서 밖으로 나가기 (`<` 키)
2. 월드맵에서 이동하며 발견되는 지역 이름 확인

### 정상 작동 예시
- **바나나 숲 아케쉬** (Banana Grove A-kesh)
- **바나나 숲 비파** (Banana Grove Bi-ppa)
- **폐허 카타라** (Ruins Ka-ta-ra)

> **✨ 체크포인트:** "Banana Grove Akesh"가 아니라 **"바나나 숲 아케쉬"**로 나와야 합니다.

## 3. 랜덤 역사/전설 (HistorySpice.json)

게임 내 책, 조각상, 벽화, 그리고 "Look" 설명에 나오는 랜덤한 역사적 사건입니다.

### 테스트 방법
1. 콘솔(`) 열기
2. `wish:spawn SultanShrine` 입력 (성소 소환)
3. 소환된 조각상을 `L` (Look) 키로 조사
4. 설명 텍스트(Description) 읽기

### 정상 작동 예시
> "이 조각상은 전설적인 **유리공예가** **파유멧**의 모습을 담고 있습니다. **1025년**, **파유멧이** **거울을** 깨뜨려 **크롬 아치**를 무너뜨렸을 때의 장면입니다."

> **✨ 체크포인트:**
> 1. 문장 구조가 자연스러운가? ("=name=(이)가" 형태가 아님)
> 2. 조사가 앞단어 받침에 맞춰 변했는가? (파유멧**이**, 거울**을**)

## 4. 랜덤 아이템 (Relics)

일반 아이템 외에, "술탄의 유물" 같은 랜덤 아이템 이름을 확인합니다.

### 테스트 방법
1. 콘솔(`) 열기
2. `wish:artifact` 입력 (랜덤 유물 생성)
3. 인벤토리(`i`)에서 이름 확인
4. 아이템을 떨어뜨리고(`d`) 다시 줍기(`g`) -> 로그 확인

### 정상 작동 예시
- 이름: **유리공예가의 검** / **빛나는 방패** / **파유멧의 수정**
- 로그: **"빛나는 방패를 획득했다!"**

---

## 🚨 문제 해결

**Q. 이름이 여전히 영어로 나와요 (예: "Fa-yu-met").**
*   `Naming.xml` 파일이 제대로 로드되지 않은 것입니다.
*   게임 재시작 후 모드 설정을 확인하세요.

**Q. 조사가 "파유멧(이)가" 이렇게 괄호로 나와요.**
*   Harmony 패치가 작동하지 않는 것입니다.
*   `Log`를 확인하여 `[Korean Josa]` 메시지가 뜨는지 확인하세요.

**Q. 지역 이름이 "Banana Grove 아케쉬" 처럼 반만 한글이에요.**
*   `Naming.xml` 의 `Template` 부분이 수정되지 않은 경우입니다. (현재 파일에는 "바나나 숲 *Name*"으로 수정되어 있으므로 정상 작동해야 함)
