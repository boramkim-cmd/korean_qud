# QA 및 품질 검증 가이드 (QA Guide)

이 문서는 한글화 프로젝트의 품질을 보장하기 위한 상세 QA(Quality Assurance) 프로세스와 검증 체크리스트를 정의합니다.

---

## 🔄 QA 프로세스 (Step-by-Step)

번역 작업 완료 후, 다음 3단계 프로세스에 따라 검증을 수행해야 합니다.

### 1단계: 기술적 검증 (Technical QA)
배포 전 자동화 도구를 사용하여 기술적 오류를 먼저 걸러냅니다.

1.  **통합 도구 실행**: 
    ```bash
    python3 tools/project_tool.py
    ```
    - 코드 구문 오류, JSON 중복 키, 번역 누락 여부를 한 번에 확인합니다.
2.  **로그 기반 미번역 탐색**:
    ```bash
    python3 tools/check_logs_for_untranslated.py
    ```
    - 실제 게임 플레이 로그(`Player.log`)를 분석하여 번역되지 않은 영어 텍스트를 찾아냅니다.
3.  **XML-Glossary 매칭 확인**:
    ```bash
    python3 tools/check_xml_glossary_match.py
    ```
    - XML 파일 내의 텍스트가 용어집(`glossary_*.json`)에 올바르게 정의되어 있는지 확인합니다.

### 2단계: 언어적 검증 (Linguistic QA)
문맥의 자연스러움과 용어의 일관성을 검토합니다.

1.  **용어 일관성**: `11_LOC_GLOSSARY_GUIDE.md`의 규칙과 JSON 데이터가 일치하는지 확인합니다.
2.  **조사 처리**: `{{item}}{을/를}` 등 한국어 조사가 모든 상황(받침 유무)에서 자연스러운지 검토합니다.
3.  **문체 확인**: `13_LOC_STYLE_GUIDE.md`에 정의된 캐릭터별 말투(존댓말/반말)가 유지되는지 확인합니다.

### 3단계: 인게임 검증 (In-Game QA)
실제 게임 내에서 텍스트가 어떻게 출력되는지 최종 확인합니다.

1.  **모드 적용**: 수정된 파일을 게임 모드 디렉토리에 복사합니다.
2.  **Wish(치트) 활용**: 
    - `Enter` 누르고 `wish` 입력 후 다음 명령어들을 사용하여 빠른 테스트가 가능합니다.
    - `item:[아이템명]`: 특정 아이템 소환하여 설명 확인
    - `goto:[지역명]`: 특정 지역으로 즉시 이동하여 NPC 대화 확인
    - `stat:Experience:10000`: 레벨업 메시지 및 스킬창 확인
3.  **로그 모니터링**: 게임 실행 중 실시간 로그를 확인하며 에러 여부를 체크합니다.
    ```bash
    tail -f ~/Library/Logs/Freehold\ Games/CavesOfQud/Player.log
    ```
