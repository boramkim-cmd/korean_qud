# 스타일 가이드

> 이 문서는 번역 스타일과 한국어 표현 규칙을 정의합니다.

---

## 기본 원칙

| 원칙 | 설명 |
|------|------|
| **일관성** | 동일 용어는 항상 동일 번역, 용어집 준수 |
| **명확성** | 게임 정보 명확 전달, 모호한 표현 지양 |
| **자연스러움** | 한국어 문법 준수, 직역보다 의역 |

---

## 존댓말 vs 반말

| 상황 | 문체 | 예시 |
|------|------|------|
| UI 텍스트 | 존댓말 | "게임을 저장하시겠습니까?" |
| 시스템 메시지 | 존댓말 | "저장이 완료되었습니다." |
| 예의바른 NPC | 존댓말 | "어서 오십시오, 여행자님." |
| 친근한/적대적 NPC | 반말 | "뭐 필요한 거 있어?" |
| 설명 텍스트 | 평서체 | "고대의 유물. 알 수 없는 에너지가 느껴진다." |
| 아이템 설명 | 평서체 | "날카로운 금속 칼날." |

## 조사 처리

### 플레이스홀더 사용

```
{{item}}{을/를} 획득했습니다.
{{npc}}{이/가} 말했다.
{{place}}{으로/로} 이동합니다.
```

### 조사 목록

| 조사 | 용도 | 받침 있음 | 받침 없음 |
|------|------|----------|----------|
| `{이/가}` | 주격 | 이 | 가 |
| `{을/를}` | 목적격 | 을 | 를 |
| `{은/는}` | 보조사 | 은 | 는 |
| `{와/과}` | 접속 | 과 | 와 |
| `{으로/로}` | 부사격 | 으로 | 로 |
| `{이다/다}` | 서술격 | 이다 | 다 |

### 색상 태그 내부 조사

```
// QudKREngine이 자동 처리
// 태그 내부의 마지막 한글로 받침 판별

{{w|검}}{을/를} → {{w|검}}을
{{y|사과}}{을/를} → {{y|사과}}를
```

## 태그 및 변수 처리

### ✅ 해야 할 것

1. **색상 태그 유지**:
   ```
   원본: {{w|Caves of Qud}}
   번역: {{w|Caves of Qud}}  // 게임 이름은 번역 안함
   
   원본: {{c|+5}} damage
   번역: 피해 {{c|+5}}
   ```

2. **변수 유지**:
   ```
   원본: =player.name= attacks!
   번역: =player.name={이/가} 공격합니다!
   ```

### ❌ 하지 말아야 할 것

1. **번역문에 태그 포함 금지**:
   ```json
   // ❌ 잘못됨
   {"{{c|ù}} text": "{{c|ù}} 번역문"}
   
   // ✅ 올바름 - 엔진이 자동 복원
   {"{{c|ù}} text": "번역문"}
   ```

2. **특수 변수 번역 금지**:
   ```
   // ❌ 절대 번역하지 말 것
   %creature%
   {{$name}}
   =item.name=
   {{hotkey|...}}
   ```

## 길이 제약

| 영역 | 최대 길이 | 비고 |
|------|-----------|------|
| UI 버튼 | 10자 | 짧고 명확하게 |

| 툴팁 | 50자 | 한 줄 권장 |

## 피해야 할 표현

### 불필요한 주어

```
❌ "당신은 레벨 업 했습니다"
✅ "레벨이 올랐습니다"

❌ "당신은 아이템을 획득하셨습니다"
✅ "아이템을 획득했습니다"
```

### 외래어 남용

```
❌ "HP가 full입니다"
✅ "HP가 가득 찼습니다"

❌ "attack 성공"
✅ "공격 성공"
```

### 어색한 존경어

```
❌ "아이템을 획득하셨습니다" (과잉 존경)
✅ "아이템을 획득했습니다"

❌ "레벨 업 하셨습니다"
✅ "레벨이 올랐습니다"
```
