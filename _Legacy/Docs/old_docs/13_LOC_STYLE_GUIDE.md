# 한글화 스타일 가이드

이 문서는 Caves of Qud 한글화 프로젝트의 번역 스타일 규칙을 정의합니다.

## 📐 기본 원칙

### 1. 일관성 (Consistency)
- 동일한 영문 용어는 항상 동일한 한글로 번역
- 용어 가이드(11_LOC_GLOSSARY_GUIDE.md) 및 로컬라이제이션 JSON 파일 준수 필수
- 캐릭터별 말투 일관성 유지

### 2. 명확성 (Clarity)
- 게임 플레이에 필요한 정보를 명확히 전달
- 모호한 표현 지양
- UI 공간 제약 고려

### 3. 자연스러움 (Naturalness)
- 한국어 문법에 맞는 자연스러운 표현
- 직역보다는 의역 우선
- 게임 분위기에 맞는 어조

## 🗣️ 존댓말 vs 반말

### UI 텍스트
**존댓말 사용** (플레이어에게 직접 말하는 경우)
```
✅ "게임을 저장하시겠습니까?"
✅ "아이템을 획득했습니다."
✅ "레벨이 올랐습니다!"
```

### 대화 텍스트
**캐릭터 성격에 따라 선택**

**존댓말** (예의 바른 NPC, 상인, 귀족)
```
✅ "어서 오십시오, 여행자님."
✅ "무엇을 도와드릴까요?"
```

**반말** (친근한 NPC, 동료, 적대적 NPC)
```
✅ "뭐 필요한 거 있어?"
✅ "꺼져, 이방인."
```

### 설명 텍스트
**평서체 사용** (아이템 설명, 스킬 설명 등)
```
✅ "고대의 유물. 알 수 없는 에너지가 느껴진다."
✅ "적에게 화염 피해를 입힌다."
```

## 🎨 태그 및 변수 처리 (Advanced)

Caves of Qud 엔진에서 사용하는 특수 태그를 올바르게 유지해야 합니다.

### 1. 색상 태그 (XML Markup)
- `{{color_char|text}}` 형식을 유지합니다.
- ✅ `{{w|검정}}`, `{{y|금색}}` 처럼 텍스트 부분만 번역합니다.
- 주의: 태그 자체를 지우면 UI에서 색상이 사라집니다.

### 2. 스마트 변수 (Engine Variables)
- `=player.name=` 또는 `{{$name}}` 같은 변수는 **절대 번역하지 않습니다.**
- 변수 뒤에 조사가 올 경우 플레이스홀더를 붙여 사용합니다.
- ✅ `=item.name={{item}}{을/를}`

### 3. 로컬라이제이션 JSON 키 매핑
- `glossary_*.json`에 정보를 추가할 때는 **camelCase**를 권장합니다.
- 키 이름은 원문 정보를 유추할 수 있도록 구체적으로 작성합니다.
- 예: `startNewGame`, `confirmDeleteSave`

## 🔧 조사 처리

### 플레이스홀더 사용
변수 뒤에 오는 조사는 플레이스홀더 사용
```
✅ "{{item}}{을/를} 획득했습니다."
✅ "{{npc}}{이/가} 말했다."
✅ "{{place}}{으로/로} 이동합니다."
```

### 조사 목록
```
{이/가} - 주격 조사
{을/를} - 목적격 조사
{은/는} - 보조사
{와/과} - 접속 조사
{으로/로} - 부사격 조사
{이다/다} - 서술격 조사
{이여/여} - 호격 조사
{이랑/랑} - 접속 조사 (구어체)
```

### 예시
```
"물{을} 마셨다." (받침 O)
"빵{을} 먹었다." (받침 O)
"사과{를} 먹었다." (받침 X)
```
