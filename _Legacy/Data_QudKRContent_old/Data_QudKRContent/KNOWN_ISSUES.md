## 과거 프로젝트에서 발견된 문제 (참고용)

자세한 내용은 [bug_analysis.md](file:///Users/ben/.gemini/antigravity/brain/45cec7a6-188c-4981-95fb-a8a7f17bb8f0/bug_analysis.md) 참조

### 1. 딕셔너리 중복 정의
- **증상**: 컴파일 에러
- **해결**: 파일 분리, 하나의 딕셔너리는 하나의 파일에만

### 2. 전역 패치 범위 문제
- **증상**: 의도하지 않은 번역 발생
- **해결**: Stack 기반 Scope 관리, 화면별 패치

### 3. 원본 메서드 확인 부족
- **증상**: Harmony 패치 적용 실패
- **해결**: AccessTools로 메서드 검증 후 패치

### 4. 데이터 중복
- **증상**: 같은 번역이 여러 딕셔너리에 존재
- **해결**: 계층 구조 사용 (Common + 전용)

### 5. 색상 태그 처리 불완전
- **증상**: `{{w|text}}` 형식 번역 안됨
- **해결**: 정규식으로 Qud 형식 처리

## 문제 보고 방법

새로운 문제 발견 시:

1. `Docs/BugReports/YYYY-MM-DD_[제목].md` 파일 생성
2. 다음 템플릿 사용:

```markdown
# Bug Report: [제목]

## 발생 일시
YYYY-MM-DD

## 증상
- 무엇이 잘못되었는가?

## 원인
- 근본 원인은 무엇인가?

## 해결 방법
- 어떻게 해결했는가?

## 예방 조치
- 재발 방지 방법

## 관련 파일
- [파일명](경로)
```

3. 이 문서(`KNOWN_ISSUES.md`)에 요약 추가
4. `CHANGELOG.md`에 변경사항 기록
